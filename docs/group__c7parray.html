Detailed Description

ポインタの配列

c7parray は記述子をインデックスとするデータの管理など、要素がとびとびになる ケースを扱う伸長可能な配列である。このようなポインタの配列を parray と呼ぶことにする。

parray の指定インデックスに新規要素を確保する。

Parameters
parrayparrayオブジェクト。
index要素を確保したいインデックス。このインデックスに要素が存在してはいけない。
Returns
処理が成功すればインデックスに新規に確保された要素のアドレスを戻し、 そうでなければ NULL を戻す。
  • インデックスに既に要素が存在した場合はエラーとなる。
  • メモリの確保に成功した場合、c7_parray_create() で init が指定されていれば、 確保したメモリアドレスと index を与えて init を呼びだす。
  • init が呼ばだされた場合、init() gal C7_FALSE を戻せばエラーとなる。この時、確保したメモリは解放される。

c7_bool_t c7_parray_move ( c7_parray_t  parray,
int  src_index,
int  dst_index,
c7_bool_t  overwrite 
)

parray の要素を別のインデックスへ移動する。

Parameters
parrayparrayオブジェクト。
src_index移動したい要素の存在するインデックス。
dst_index移動先のインデックス。
overwrite移動先のインデックスに要素があった場合に上書きするかどうかを指定する。 C7_TRUE であれば c7_parray_free() でその要素を削除し、C7_FALSE であればエラーとする。
Returns
移動に成功すれば C7_TRUE を、失敗すれば C7_FALSE を戻す。

成否は次のように定まる:

  1. src_index と dst_index が同じ場合は何もせずに成功。
  2. dst_index に要素が既に存在して overwrite が C7_FALSE であれば失敗。
  3. dst_index に要素が既に存在して overwrite が C7_TRUE であれば成功。
    • c7_parray_create() で deinit が定義されていれば dst_index に対して呼ばれる。
    • 移動完了後、src_index は空きとなる。
