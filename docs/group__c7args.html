Detailed Description

コマンドライン引数のオプション解析

コマンドライン引数のオプションを解析するには、 各オプションの特徴を構造体で定義し、 オプション検出時に呼ばれるハンドラを準備することから始まる。 それを解析機に登録したあと、実際の引数(いわゆる main() の argv)を 渡すことで解析が行われる。

Usage表示用の文字列を得る。

Parameters
apオプション解析オブジェクト。
sbpUsage文字列を追記するC7文字列。
expl_offオプションの概要(optv[].optrepr)を表示するカラム位置。
Returns
sbpを戻す。

オプション解析オブジェクト ap から Usage 表示用の文字列を構成し、C7文字列 sbp に追加する。

以下のような文字列を得ることができる。

-c, --category=CATEGORY[,...]
print only specified category
CATEGORY: category (0..31)
--clear clear contents after print
-d, --date=DATETIME[,DATETIME]
range of date time
DATETIME: timestamp: [[[YY]MMDD]hhmm][.ss] or '.'
-g, --level=LOG_LEVEL maximum log level
LOG_LEVEL: log level (0..7)
-h, --help this help
-r, --record=MAX_COUNT maximum count of record
MAX_COUNT: count record
-s, --order=ORDER[,ORDER] range of order
ORDER: order of record
-t, --thread=THREAD_ID print only specified thread
THREAD_ID: thread id

この文字列の中で、例えば "CATEGORY[,...]" の "CAGEGORY" は optv[].prmword で指定した文字列、"[,...]" や "[,DATETIME]" などは optv[].prmc_min, optv[].prmc_max の情報をもとに生成される。

char** c7_args_parse ( c7_args_t  ap,
char **  argv,
void *  __uctx 
)

引数を解析しハンドラを呼び出す。

Parameters
apオプション解析オブジェクト。
argv解析する引数。argv[0] は最初の引数でなければならない。
__uctxハンドラに渡される引数。
Returns
オプションでない文字列 argv[n] を検出した時の &argv[n] を戻す。
  • 解析できたオプションに対して、オプション定義に指定したハンドラが呼び出される。
  • ハンドラの params 引数に渡された解析結果のメモリは、同一の解析オブジェクト ap に対して次に c7_args_parse() を呼び出すか c7_args_free() を呼び出すと解放される。 そのため、解析結果の文字列などを永続的に使用したい場合は呼び出し側で複製する必要がある。
  • オプションでない引数を検出したらそこで解析を停止する。
